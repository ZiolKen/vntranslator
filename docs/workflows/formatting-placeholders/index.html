Formatting & placeholders

Most machine translation failures in VN/RPG projects are not “bad translation” — they are formatting breakage.

This page lists what you should preserve and how to QA it.

Ren’Py essentials

Text tags

Keep tags exactly and ensure they close:

  • {i}...{/i}
  • {b}...{/b}
  • {color=#fff}...{/color}
  • {size=+10}...{/size}

Variables / interpolation

Do not change anything inside [...]:

  • [player_name]
  • [points]

Escape sequences

Preserve escapes like:

  • \n
  • \"

RPG Maker essentials (MV/MZ)

Control codes (examples)

  • \V[1] variable value
  • \N[3] actor name
  • \C[2] color
  • \I[10] icon
  • \G currency
  • \{ \} text size

JSON escaping

RPG Maker strings are inside JSON, which uses escapes:

  • a single backslash often becomes \\ in JSON text
  • line breaks may be \n or \\n depending on context

If you break escaping:

  • the JSON becomes invalid
  • the game may fail to boot

QA patterns that catch most issues

  • Compare placeholders before/after:
    • Ren’Py: count { and }
    • RPG Maker: search for \V[ and ensure it still exists
  • Search for broken sequences:
    • {/i missing }
    • \V[ changed to V[ (lost backslash)
  • Use Preview tools to visually confirm message windows

If you’re unsure

  • use smaller batch sizes
  • translate fewer lines
  • fix manually in editor

This is slower but dramatically safer for production builds.
