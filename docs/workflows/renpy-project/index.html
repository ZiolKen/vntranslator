Translate a Ren’Py project

This workflow is optimized for stability and minimal “random engine errors”.

0) Start with the source project

Open the game in Ren’Py Launcher and confirm you can run it.
Do not start from a random build if you can avoid it.

1) Backup

  • Zip the whole project folder, or
  • initialize a git repo

2) Translate safely

Option A: Single file flow

  1. Open: Ren’Py Translator
  2. Upload a .rpy file (from game/)
  3. Translate, then QA in Preview
  4. Export and replace the file in the project

Option B: Project flow (recommended)

  1. Open: Pro Ren’Py Translator
  2. Open the project folder
  3. Translate missing/untranslated strings
  4. Fix warnings and export ZIP

3) QA checklist

  • tags are balanced ({i} → {/i})
  • variables unchanged ([player_name])
  • no broken quotes
  • indentation preserved

See: Formatting & placeholders

4) Test in Ren’Py

  • Run from Ren’Py Launcher
  • Click through scenes that contain:
    • formatted text
    • menus
    • UI screens

If you hit an error:

  • read the traceback (it usually points to a line in .rpy)
  • fix the broken line
  • export again

5) Iterate

For VN projects, translation is iterative:

  • translate → QA → test → adjust terminology → repeat
