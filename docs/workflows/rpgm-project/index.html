Translate an RPG Maker project (MV/MZ)

This workflow focuses on keeping JSON valid and preserving RPG Maker control codes.

0) Start from a working project

Open the project in RPG Maker and confirm it runs.

1) Backup

Zip the project folder (at minimum: data/).

2) Choose a translation approach

Option A: Single JSON flow

Use RPG Maker Translator when you translate one file at a time:

  • System.json (menus/system strings)
  • CommonEvents.json
  • MapXXX.json

Option B: Multi-file flow (recommended)

Use RPGM Ultimate for many files:

  • export TXT packages
  • translate in batches
  • import back to JSON

3) Preserve control codes

RPG Maker message strings contain control codes like:

  • \V[1], \N[3], \C[2], \I[10]

Breaking these often causes:

  • missing names
  • broken UI
  • visual glitches

See: Formatting & placeholders

4) QA and test

  • Replace JSON files back into data/
  • Playtest in RPG Maker
  • Test:
    • menus
    • shops
    • battle messages
    • events and maps

If the game refuses to load:

  • the JSON may be invalid (quote/backslash issue)
  • re-open the file in editor and fix escaping
