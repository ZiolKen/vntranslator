Ren’Py Translator

Use this tool to translate Ren’Py .rpy scripts while preserving VN-specific formatting.

Open the tool: Ren’Py Translator

When to use it

  • You have one .rpy file you want to translate now
  • You want a simple workflow: upload → translate → preview → export

If you want multi-file project translation with Translation Memory, use: Pro Ren’Py Translator

Inputs

  • .rpy script files from the Ren’Py project source
  • Usually located under: game/*.rpy

Guide: How to get .rpy files

Step-by-step

  1. Open Ren’Py Translator
  2. Choose a translation engine:
    • DeepSeek / DeepL / Lingva (depending on tool UI)
  3. Select your target language
  4. Upload the .rpy file
  5. Start translation (begin with a small batch size if available)
  6. Open Preview & Editor to QA and fix issues
  7. Export the translated .rpy and put it back into your game/ folder
  8. Run the project and test

Ren’Py formatting rules (must preserve)

Tags

Examples:

  • {i}italic{/i}
  • {b}bold{/b}
  • {color=#fff}white{/color}

Tags must remain balanced.

Variables / interpolation

Examples:

  • [player_name]
  • [points]

Do not translate or rename variables.

Escapes

Examples:

  • \n (line break)
  • \" (quote)

Keep escapes valid.

Preview & Editor (recommended QA)

Preview helps you catch:

  • broken tags (missing {/i})
  • corrupted variables ([player] → [playerName])
  • extra quotes or missing braces
  • awkward line breaks

If you see a broken line:

  • fix it in the editor
  • re-export
  • test again

Dialog Extractor & Merge (advanced)

If you want a safer “dialogue-only” workflow:

  • extract dialogues from .rpy
  • translate only extracted lines
  • merge back into the original script

Tool page: Ren’Py Dialog Extractor

Best practices

  • Translate from source, not from random builds
  • Back up before replacing scripts
  • Translate one file first, test, then continue
